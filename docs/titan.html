Strategy Overview

Objective: Generate ~50% CAGR with controlled drawdowns on crypto, executable manually on OKX (leveraged tokens) or Hyperliquid (perpetuals). Long-only. Weekly rebalance Wednesdays. Survivorship-bias-aware universe of 50 coins (35 survivors + 15 dead/collapsed).

How it works: 21 weakly-correlated sleeves vote on each coin's target weight. The aggregated weights are sleeve-vol-targeted, RP-blended, consensus-gated, and weekly-snapped. A daily vol-target overlay scales total exposure. A conviction-scaled leverage curve (1×→3.2×) amplifies high-agreement periods. A vol-of-vol regime gate cuts leverage when vol is expanding. A self-stop cuts size on drawdowns over -4% in 30 days.

Sleeve families:

  • Trend core (4): BTC_VM (fast vol-managed), BTC_SLOW (200MA + 126d mom), TRIPLE_MOM (21/63/252 confluence), MTF_FRACTAL (NOVEL: long only when ALL 6 timeframes 5d/21d/63d/126d/252d/365d positive simultaneously)
  • Breakout (3): TURTLE20, TURTLE55 (Donchian breakouts), BOLLINGER (2σ upper-band breakouts), CONSOLIDATION_BREAK (NOVEL: 40d-tight-base breakout), CONVEX_BREAKOUT (NOVEL: NO vol-targeting → preserves right-tail convexity, trailing stops only)
  • Regime / macro (3): MACRO_TREND (SPY+VIX gated BTC), HALVING_BOOST (post-halving deterministic window), AUTOCORR_REGIME (long only when 63d lag-1 autocorr positive)
  • Specialist (2): CRASH_DIP (buy -20→-40% DD in uptrend), VOL_BREAKOUT (low-vol regime breakouts)
  • Cross-asset (1): ETH_VM
  • Proprietary edges: EFFICIENCY_TREND (Kaufman ER-sized trend), RANGE_EXPANSION (range×close-position), INV_VOL_BTC (Moreira-Muir), VOL_PERCENTILE (size by vol percentile rank), BREADTH_THRUST (Zweig kickoff), SPY_ALPHA (BTC beating beta-adj equity hurdle)

Portfolio overlays (1-day lagged):

  1. Risk-parity sleeve weights: inverse 90d realised vol of each sleeve's returns.
  2. Consensus power-curve: scale = ((conv − 0.15)/0.55)1.1 × 1.35; below 0.15 sleeve agreement, full cash.
  3. Master gate: weighted vote over BTC>200MA, BTC DD>-30%, BTC 63d-mom>-8%, breadth>25%; full cash if score < 0.25.
  4. Weekly snap: positions held constant Wed-to-Wed; only re-evaluated weekly.
  5. Daily vol target: EWMA(60d) realized vol → 18% target, asymmetric clip [0.3, 1.5].
  6. Daily DD floor: linear scale-down from 0% to -12% NAV drawdown vs 365d HWM.
  7. Self-stop: NOVEL — when own 30d return < -4%, halve exposure for 21 days.
  8. Catastrophic-DD per coin: any coin in >28% DD from 60d high → forced to 0 weight.
  9. Conviction-scaled leverage: 1× base, ramps to 3.2× when sleeve agreement crosses 0.85.
  10. Vol-of-vol regime gate: NOVEL — when 14d-smoothed vol has risen >20% over 28d, cut leverage extra by 30%.

The 50-coin set already includes 30% explicitly-dead coins. The bias-delta test runs against all 111 to confirm robustness — strategy SR improves when dead coins are explicitly included (no survivorship bias).

Honest assessment — across 8 iterations:

After v1 → v8 (each commit on the feature branch is preserved), CRYPTO-TITAN reaches Full SR 1.47, OOS SR 1.13, CAGR 51.4%, MDD -32%, NAV 124× over 11.5 years. CAGR target (50%+) is met via conviction-scaled leverage on Hyperliquid perps / OKX leveraged tokens.

OOS Sharpe plateau is real: across all configurations tested (universe sizes 20→111, sleeve counts 4→23, daily vs weekly rebal, leverage 1× to 3.5×, several short variants, multiple regime gates and conviction curves), OOS Sharpe stays in the 1.10-1.50 range. The published crypto-quant ceiling for long-only daily spot bars without funding-rate or options data is ~SR 2.

Empirical findings worth flagging:
  • Shorts don't work on daily bars in this sample. Tested SHORT_BTC_BEAR, SHORT_TREND_BREAK, SHORT_PERMABEAR, LONG_SHORT_XSMOM. All -SR. Bounce-squeeze + funding drag + structural BTC uptrend over 11 years punishes systematic short bias. Daily-bar shorting needs intraday data + funding signals.
  • Naive cross-sectional XSMOM across 100+ coins blows up (-60% MDD). Picks "trending" alts right before they crash. Survivorship-bias test passes (no false alpha) but signal is too noisy.
  • Asset concentration kills — going top-K per day with leverage amplified losses too. Time-concentration (cash when uncertain, lever when sure) was the better interpretation.
  • Calendar anomalies inverted post-2022. Turn-of-month had OOS SR -0.63. Day-of-week effects are now noise.
  • Survivorship bias is NEGATIVE in this strategy — the full universe SR is 0.10 HIGHER than survivors-only. Including dead coins makes the strategy better (the eligibility + catastrophic-DD overlay correctly excludes them before damage; their historical dispersion just adds noise to weak baselines, not to ours).

Path to genuinely higher OOS SR (3+) requires NEW DATA:
  • Perp funding rates from Hyperliquid/Bybit/OKX — long spot + short perp captures basis (cash-and-carry). Historical Sharpe 3-5. Adds one extra trade leg per coin per week.
  • Higher-frequency intraday data — orderbook microstructure, funding-rate changes, basis spreads. Sharpe 2-4 typical for tactical entries.
  • Options vol-risk-premium — sell premium on Deribit BTC options. Sharpe 2-3 typical. (User has indicated no options.)

Survivorship bias robustness (3-way test)
Test design: Run the same strategy code path against three universes — (a) SURVIVORS-only 35 coins (the classical biased view that drops failed coins); (b) STRATEGY universe 50 coins (35 survivors + 15 dead); (c) EXTENDED universe 111 coins (everything dead we have data for). The strategy's SR should be roughly equal or HIGHER on the wider universes — that is the negative-bias-delta test.

Result: Strategy SR is HIGHEST on the 50-coin set with dead coins included. Survivors-only is the LOWEST. This means the strategy's signal logic + catastrophic-DD overlay correctly exclude dead coins before they damage performance, AND the dead coins' historical correlation structure helps the ensemble. ΔSR (survivors − strategy) = -0.10 — classical survivorship bias would produce a positive delta; our strategy passes.

Live execution — manual rebalance flow
Recommended setup: Start manually on OKX with leveraged tokens for 4-8 weeks. Once the signal is proven live, port to a Hyperliquid bot for automated weekly rebalance.

Weekly Wednesday workflow (15 min)

  1. Fetch latest closing prices into data/crypto/ (yfinance via scripts/expand_universe.py).
  2. Run python live_signal.py --venue okx --capital <USD> after market close.
  3. Output is a target-weights table with a "trades vs last week" diff (OPEN / CLOSE / FLIP / ADD / TRIM).
  4. Execute the diff manually:
        OKX: use spot for <1.5× weights, BTC3L/ETH3L for long-leveraged, BTC3S/ETH3S for short-leveraged (currently disabled but framework ready).
        Hyperliquid: open/close/resize perp positions at the indicated leverage.
  5. Save snapshot of executed weights for next-week diff.

Hyperliquid bot path (after live validation)

  • Hyperliquid has a public Python SDK (hyperliquid-python-sdk).
  • Bot would: cron weekly Wed 00:00 UTC → fetch fresh prices → run strategy → diff vs current Hyperliquid positions → submit signed order batch.
  • Defensive overlays: per-coin position cap, daily loss-limit kill-switch, manual approval mode for first month.

Key execution costs & haircuts (already in backtest)

  • Spot taker fee: ~10 bps. Modelled at 20 bps round-trip.
  • Perp taker fee: ~5-7 bps. Modelled at 30 bps round-trip (conservative).
  • Funding rate on perp leverage: ~3 bps/day in long-heavy regimes (~15%/yr) on the levered notional above 1×. Modelled.
  • Slippage on illiquid alts: NOT modelled. Limit positions to 1% of daily volume (manual check on coins like KNC, CELR).
